# 每周组会长期工作项目规则

## 1. 适用范围

本文件所在目录是本项目的工作根目录。Codex 应在本目录内长期协助管理每周组会、阶段性任务、工作记录、成果整理和阶段归档。

处理相关任务时，应使用项目 Skill：

```text
managing-weekly-group-meetings
```

Skill 默认位置：

```text
.agents/skills/managing-weekly-group-meetings/SKILL.md
```

## 2. 权限边界

未经用户明确授权，Codex 只能读取和修改本根目录及其子目录。

不得擅自：

- 访问、扫描、修改或删除根目录外的文件；
- 安装软件、依赖、插件或驱动；
- 使用管理员权限；
- 修改系统设置；
- 上传本地文件到外部网站、云盘或第三方服务；
- 将本项目内容发送给外部服务；
- 大范围移动、覆盖或删除已有文件。

确需额外权限时，必须先说明：

1. 需要的权限；
2. 使用目的；
3. 拟访问或修改的位置；
4. 是否存在不需要额外权限的替代方案。

用户未明确同意前，不得执行。

## 3. 标签规则

用户发布阶段性任务时必须提供标签，例如：

```text
[paper]
[burnup]
[cf252]
```

同一项跨周任务必须持续使用同一标签。

若用户忘记提供标签，Codex 必须提醒用户补充。补充标签前，可以分析需求，但不得创建正式任务成果或擅自决定永久标签。

所有任务记录和正式成果文件名必须以标签开头：

```text
[label]文件名.ext
```

## 4. 目录规则

根目录结构：

```text
根目录/
├─ AGENTS.md
├─ TASKS.md
├─ weekly/
│  └─ YYYY-Www/
│     ├─ README.md
│     ├─ tasks/
│     ├─ temporary/
│     └─ deliverables/
├─ completed-projects/
│  └─ [label]阶段任务名称/
│     ├─ README.md
│     ├─ source-index.md
│     └─ deliverables/
└─ .agents/
   └─ skills/
      └─ managing-weekly-group-meetings/
```

临时文件必须放入当周的 `temporary/`，重要成果必须放入当周的 `deliverables/`。

## 5. 全局任务清单

Codex 必须持续维护根目录下的：

```text
TASKS.md
```

其中记录所有阶段性任务的：

- 起始日期；
- 任务标签；
- 任务名称；
- 大致描述；
- 当前状态；
- 完成日期；
- 实际执行周次；
- 最近更新日期；
- 当前记录或归档位置。

任务进度发生变化时，应在本次工作结束前同步更新 `TASKS.md`。

## 6. 每项工作记录

每周的每一项实际工作都必须有独立 Markdown 记录，放入：

```text
weekly/YYYY-Www/tasks/[label]任务名称.md
```

记录应包含目标、输入、过程、判断、临时文件、正式成果、当前结果、未解决问题和下一步。

开始相同标签的新工作前，应查看历史周次的相关 Markdown 记录，不得只依赖聊天记忆。

## 7. 阶段归档

只有用户明确说明阶段任务完成时，才执行阶段归档。

归档采用复制方式：

- 将相关正式成果复制到 `completed-projects/[label]阶段任务名称/`；
- 保留 `weekly/` 中的原文件；
- 不移动或删除每周历史记录；
- 不归档无必要的临时文件；
- 建立阶段总结和来源索引。

## 8. 周报和组会材料

用户要求整理本周组会、生成文档或 PPT 时，应根据：

- 本周 README；
- 本周任务记录；
- 本周正式成果；
- 必要的历史记录；

整理汇报材料，并将正式文件放入当周 `deliverables/`。

## 9. 文件安全

修改现有文件前先读取。优先局部修改，不得无故覆盖正式成果。

存在同名成果时，默认创建版本号：

```text
[label]report-v1.docx
[label]report-v2.docx
```

只有用户明确要求覆盖时才可覆盖。

## 10. 首次初始化

首次运行时，可以创建缺失的：

- `TASKS.md`
- `weekly/`
- `completed-projects/`

不得自动移动、重命名或清理根目录中的已有文件。
